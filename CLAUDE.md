## Project Overview

This is an FPGA FFT (Fast Fourier Transform) signal processing design project targeting Xilinx FPGAs via Vivado 2024.2. It implements a 256-point FFT using the Xilinx FFT IP core, processes complex 12-bit input signals at 200 MHz, and detects the peak frequency bin.

**Key parameters:**
- Sampling frequency: 200 MHz
- FFT length: 256 samples
- Input word width: 12 bits (fixed-point, Q1.11)
- Bin resolution: 781.25 kHz per bin
- Test tones: 12.5 MHz (exact → bin 16), 10 MHz (leaks → bin 13)

## Workflow & Commands

### Python (Signal Generation & Reference)
```bash
# Generate test tone memory files and verify expected FFT bins
python python/reference_fft.py

# Generate Hamming window coefficients (Q1.15 fixed-point)
python python/generate_hamming_mem.py
```

### Vivado (Simulation & Synthesis)
Open Vivado 2024.2, then in the Tcl Console:
```tcl
# Add RTL and testbench sources to the project
source scripts/add_part1_sources.tcl

# Run simulation
launch_simulation
```
Or open `fpga_fft_project/fpga_fft_project.xpr` directly in Vivado.

## Architecture

### RTL Modules (`rtl/`)
- **`fft_naive_top.v`** — Top-level wrapper. Instantiates the Xilinx `xfft_0` IP. Packs 12-bit real and imag inputs into the AXI4-Stream 32-bit data bus (`{imag[11:0], real[11:0]}` at bits 27:16 and 11:0), drives FFT configuration (forward, no scaling), and connects output to the magnitude/peak pipeline.
- **`abs_sq.v`** — Computes magnitude-squared (real² + imag²) from FFT output samples for each bin.
- **`peak_detector.v`** — Iterates over the 256-bin FFT output frame and outputs `peak_index` and `peak_value` for the maximum magnitude-squared bin.

### Testbench (`tb/`)
- **`tb_fft_naive.sv`** — SystemVerilog testbench. 200 MHz clock. Loads `tone_12p5MHz.mem` and `tone_10MHz.mem`, streams two FFT frames via AXI4-Stream, and asserts expected peak bins (16 and 13 respectively).

### Memory Files (`mem/`)
- `tone_12p5MHz.mem` — 256 complex samples, 12-bit real + 12-bit imag (generated by Python)
- `tone_10MHz.mem` — 256 complex samples (spectral leakage expected)
- `hamming_q1_15.mem` — 256 Hamming window coefficients in Q1.15 format

### Python Utilities (`python/`)
- `reference_fft.py` — NumPy FFT reference; generates `.mem` files and prints expected peak bins
- `generate_hamming_mem.py` — Converts Hamming window to Q1.15 hex and writes `hamming_q1_15.mem`

### Xilinx IP
- `xfft_0` — 256-point, 12-bit input, AXI4-Stream interface, configured for forward FFT

## AXI4-Stream Interface Notes

The Xilinx FFT IP uses AXI4-Stream. Key signals:
- `s_axis_data_tdata[31:0]`: input packed as `{4'b0, imag[11:0], 4'b0, real[11:0]}`
- `s_axis_data_tvalid` / `s_axis_data_tready`: handshake
- `s_axis_data_tlast`: marks last sample of a 256-sample frame
- `m_axis_data_tdata[63:0]`: output (imag in upper 32 bits, real in lower 32 bits, sign-extended)
- `m_axis_data_tlast`: marks end of output frame
